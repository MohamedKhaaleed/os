The Interface

System Calls

System calls provide the low-level interface between a running program and the Operating System. They act as the bridge between User Mode and Kernel Mode.

  • Protection: Ensures processes cannot interfere with each other or access unauthorized hardware directly.
  • Abstraction: Programs use high-level APIs (like POSIX or Windows API) which internally trigger these calls for portability.
  • Mechanism: Triggered via interrupts/traps, switching control to the kernel to execute the privileged operation.
